생기부 작성기 사용자 가이드

소개

생기부 작성기는 만세진이 교사용으로 만든 생활기록부 초안 작성 도구입니다. 공개 도구함에는 노출되지 않으며, 관리자 비밀번호로 로그인한 교사만 사용할 수 있습니다.

교사가 직접 할 일은 크게 네 가지입니다.

  1. 예전에 작성한 생기부 샘플을 올려 문체·분량 패턴을 학습시킵니다.
  2. 학생별 활동 메모(행발·세특·창체)를 등록합니다.
  3. 영역별로 AI 일괄 작성을 실행합니다.
  4. 결과를 검사·수정한 뒤 NEIS에 옮깁니다.

참고

이 도구는 NEIS에 자동으로 붙여 넣지 않습니다. 작성·검토 후 교사가 직접 생활기록부 시스템에 입력해야 합니다.

제작 배경

생활기록부는 학생마다 비슷한 형식이지만, 문장 톤·분량·강조점은 담당 교사의 스타일을 반영해야 합니다. 매 학기 수십 명의 행발·세특·창체를 처음부터 쓰는 데는 시간이 많이 들고, 수행평가·활동 내용을 학생마다 반복 입력하는 작업도 부담이 됩니다.

생기부 작성기는 다음 문제를 줄이기 위해 만들어졌습니다.

  • 과거에 쓴 생기부의 문체를 매번 다시 맞추는 수고
  • 활동 메모를 화면에서 한 명씩 입력·복사하는 시간
  • NEIS 글자수(바이트) 한도를 작성 후에야 확인하는 불편
  • 초안을 여러 번 고칠 때 검토 기준이 들쭉날쭉한 점

AI가 초안을 대신 쓰는 것이 아니라, 교사의 스타일과 입력 메모를 반영한 초안을 빠르게 만든 뒤 교사가 최종 검토·수정하는 흐름을 전제로 합니다.

작동 원리

시스템은 크게 스타일 학습과 학생별 작성 두 축으로 동작합니다.

과거 생기부 샘플 → 문체·분량 분석 → 스타일 가이드 → Gemini 작성 → 검사·수정

스타일 학습

교사가 업로드한 xlsx·docx·tsv 등 샘플 파일에서 문장 길이, 자주 쓰는 표현, 항목별 분량을 분석합니다. 분석 결과는 스타일 가이드 텍스트로 저장되며, 이후 모든 AI 작성 프롬프트에 포함됩니다.

학생 데이터

학생 이름·학년·반·번호와 행발 메모, 과목별 세특(진로·수행평가·주제·활동), 창체(자율·동아리·진로) 메모가 서버에 구조화되어 저장됩니다. 일괄 작성 시 이 데이터와 스타일 가이드를 조합해 항목별 초안을 생성합니다.

작성본 보관

기본 설정에서는 AI가 만든 작성본은 브라우저에만 보관됩니다. 서버에는 학생 메모만 암호화 저장됩니다. 탭을 닫기 전 엑셀로 백업하는 것을 권장합니다.

AI 모델

모든 AI 호출(샘플 분석, 메모 파싱, 생기부 작성, 검사 보조)에 Google Gemini 3.1 Pro 단일 모델을 사용합니다.

용도설명
샘플 분석업로드한 과거 생기부에서 문체·분량 패턴 추출
생기부 작성학생 메모 + 스타일 가이드 → 행발·세특·창체 초안 생성
PII 마스킹전화·이메일·주소 등은 API 전송 전 자동 마스킹 (SGB_MASK_PII=1)

중요

학생 메모 텍스트는 작성 시 Google Gemini API로 전송됩니다. 민감 정보는 최소한만 입력하고, 학교·교육청 지침을 따르세요.

분량·글자수 규칙

NEIS 입력란 바이트 한도는 neis-counter 규칙을 따릅니다.

  • 행발 — 900바이트
  • 세특·창체 — 1,500바이트

상세 편집 화면과 검사 기능에서 공백 제외·포함 글자 수와 바이트를 함께 표시합니다.

xlsx, docx, tsv · 여러 파일 선택 가능
  1. 파일 선택 — xlsx, docx, tsv 등 여러 파일을 한 번에 선택할 수 있습니다.

  3. 분석 시작 — Gemini가 샘플을 읽고 스타일 가이드를 생성합니다. 완료까지 수십 초~수 분 걸릴 수 있습니다.

팁

샘플에는 학생 실명·주민번호 등 식별 정보를 넣지 마세요. 익명화한 예시 파일을 사용하는 것이 좋습니다.

② 스타일 설정

①에서 생성된 스타일 가이드를 확인하고, 원하는 톤으로 직접 수정할 수 있습니다. 분석이 끝나면 아래처럼 문장 규칙이 자동으로 채워지며, 저장하면 이후 모든 AI 작성 프롬프트에 포함됩니다.

화면 예시 — 스타일 가이드 (분석 후 자동 작성)

※ 실제 화면에서는 ① 분석 시작 직후 위와 같이 가이드 텍스트가 생성·표시됩니다.

예시 프롬프트에 자주 들어가는 지시: 「서술체 사용」, 「활동 나열보다 태도·성장 강조」, 「문단당 3~4문장」, 「행발·세특 분량 목표」 등. 구체적으로 적을수록 결과가 안정적입니다.

③ 학생 등록

작성할 항목을 선택한 뒤, 학생 기본 정보와 메모를 입력합니다.

작성할 항목

행발·세특·자율·동아리·진로 중 필요한 항목을 선택합니다. 선택한 항목에 해당하는 메모 입력란만 표시됩니다.

화면 예시 — 작성할 항목

학생 추가

이름·학년·반·번호를 입력하고, 선택한 항목의 메모를 작성한 뒤 학생 추가를 누릅니다.

  • 세특 — 과목명은 필수입니다. 진로·수행평가 형식·주제·활동 내용을 입력할 수 있습니다.
  • 행발 — 태도·성품·대인관계 등 관찰 메모를 적습니다.
  • 창체 — 자율·동아리·진로 활동 메모를 각각 입력합니다.

등록된 학생 목록 왼쪽 체크박스로 여러 명을 고른 뒤, 선택 삭제 또는 전체 초기화를 실행할 수 있습니다. 각 행의 메모·삭제 버튼으로 한 명씩 다룰 수도 있습니다.

기대 효과

  • 화면에서 학생별로 반복 클릭·입력하는 시간 절감
  • 수행평가 등 동일·유사 내용의 반복 복사·붙여넣기 감소
  • 엑셀에서 여러 학생을 한 번에 작성 후 일괄 반영

보내기 / 불러오기 대상

학생 기본 정보 + 입력 메모만 포함됩니다. AI 작성 완료본은 포함되지 않습니다.

사용 방법

  1. 엑셀보내기 — 현재 등록된 학생을 프로그램이 인식하는 형식으로 저장합니다.
  2. 엑셀에서 작성 — 학생 정보를 수정·추가합니다. 셀 크기는 변경해도 되지만, 열 이름(헤더)·열 구조는 변경하면 안 됩니다.
  3. 가져오기 — 저장한 파일을 선택하고 가져오기를 실행합니다.

검사 기능

검사는 다음 항목을 확인합니다.

  • NEIS 바이트 한도 초과 여부
  • 금지·주의 표현

자주 묻는 질문

행발과 세특을 한 번에 작성할 수 있나요?

④에서 영역을 선택해 각각 실행해야 합니다. 행발 작성 후 세특으로 영역을 바꿔 다시 일괄 작성을 실행하세요.

세특 작성이 안 될 때

③에서 해당 학생의 과목명이 입력되어 있는지 확인하세요. 과목명은 필수입니다.

작성본이 사라졌어요

기본 설정에서는 작성본이 브라우저에만 저장됩니다. 탭·브라우저를 닫으면 사라질 수 있습니다. 정기적으로 엑셀 추출로 백업하세요.

엑셀 가져오기가 실패해요

맨 윗줄 열 이름과 열 순서를 변경하지 않았는지 확인하세요. 이름·학년·반·번호가 비어 있는 행은 건너뛸 수 있습니다.

무료 사용 한도가 있나요?

서버 설정에 따라 월 작성 횟수 한도가 있을 수 있습니다. ④ 상단에 이번 달 작성 횟수가 표시됩니다.
